# 生命之书 LifeBook 本地演示版

这是一个可本地运行的 Web MVP demo，用于演示「家庭上传成长记录 → 本地 AI 式分析 → 成长时间线 → 陪伴建议 → 成长卡/月度报告」的核心闭环。

## 运行方式

仅体验静态 demo 时，可以在当前目录启动静态服务：

```bash
python3 -m http.server 4173
```

然后打开：

```text
http://localhost:4173
```

如果要体验真实 AI 看图分析，请使用 Node 服务端启动，避免把 API Key 暴露在前端：

```bash
cp .env.example .env
# 编辑 .env，填入 OPENAI_API_KEY
npm run dev
```

然后打开：

```text
http://localhost:4173
```

在「记录成长」里上传照片，勾选「使用 AI 看图分析」，保存后会调用 `/api/analyze`。如果没有配置 `OPENAI_API_KEY`，页面会自动降级成本地规则分析。

## 已包含能力

- 上传照片、视频、语音和文字记录
- 使用 IndexedDB 保存在当前浏览器本地
- 自动生成成长总结、成长标签、成长画像和陪伴建议
- 展示成长时间线、成长树、家庭成员贡献
- 生成周成长摘要和月度生命之书
- 导出单条记录的 PNG 成长卡
- 生成可独立打开的互动 HTML 生命之书电子绘本
- 可选择小禾、晴天、张宝心三个公开演示角色，并切换到对应角色专属的图片演示样本
- 每个演示角色内置 10 张照片，用于头像、照片墙和互动电子书
- 生命之书页面内置四个预设角色的互动电子书链接，可直接打开对应章节查看

## 说明

当前版本保留本地演示分析引擎，并新增了可选服务端 AI 图片分析接口：`server.js` 提供本地 Node 服务，`api/analyze.js` 可用于 Vercel 这类 Serverless 部署。图片会在浏览器内压缩成 JPEG 后再发送到服务端；服务端不保存图片，只把本次请求转发给大模型并返回结构化成长分析。

`app/assets/demo/` 内置图片仅用于公开 demo 演示。小禾、晴天、张宝心素材由 AI 生成，不代表真实儿童、学校或家庭数据。选择不同演示角色会替换当前演示记录，避免多个案例叠加。

`app/books/` 内置四个预设互动生命之书 HTML，引用对应演示素材，仅用于展示电子绘本式交互效果。
